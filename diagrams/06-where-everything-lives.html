Where does everything actually live?

There is no "the server." Your code is everywhere. Your data is in Virginia. That gap is the real architecture.

1 · The split that matters

⚡ Your code — everywhere

wrangler deploy copies the whole script, including the 18 KB HTML string, to every data centre Cloudflare runs.

~300
cities holding a copy of your Worker

Your page is not a file. It is a string inside code that has been replicated globally, manufactured on demand and thrown away.

🗄️ Your data — one place

D1 is not replicated to the edge. It sits in a single region, and every query goes there.

"Edge computing" does not mean fast everywhere. It means your code is close. Whether that helps depends on what the code has to reach.

3 · Inside any one machine — and what the split costs

🧩 One machine, thousands of tenants

Zoom into a single machine in Ashburn. Right now it is running thousands of isolates belonging to thousands of unrelated customers.

Each sandbox ~3 MB · starts in ~5 ms · cannot see the others' memory · most alive for milliseconds

This is the trade. Cloudflare cannot give you a machine — that would be absurd for something idle most of the time. You get a slot on a shared one. In exchange you accept 10 ms CPU, no filesystem, no memory between requests.

🌍 The same request, two places

From New Jersey 🇺🇸
  1. Worker runs in a nearby colo
  2. D1 query → Ashburn, next door
  3. AI → Workers AI
  4. ~1.2s total
From Singapore 🇸🇬
  1. Worker runs in Singapore — code is local
  2. D1 query → across the Pacific to Virginia
  3. and back again
  4. Noticeably slower, and the code did nothing wrong

You sit beside your own database, so you never feel this. A user on the other side of the world would. The fix is read replicas or moving the data — not faster code.

4 · "Cached" vs "stored" — the distinction that decides what can break

🗂️ A normal CDN — this IS caching

A temporary copy of something that lives somewhere else.

edge cache (temporary)
  ↓ miss
your server (the truth)
  ↑ if this is down, you are down

The origin is the real home. The edge just holds a copy for a while.

📦 Pages and Workers — this is NOT caching

Built once from GitHub, then uploaded and replicated. There is no origin behind it.

edge (the truth, everywhere)
  ↑ there is no "behind"
nothing to fall back to
nothing to be down

This is why static sites on Pages are hard to break. A cache miss cannot hurt you, because a miss is not possible in the same way.

D1 is the exception — it genuinely lives in one place, which makes it the only part of the stack that behaves like a traditional origin. Caching does still happen on top of all this (AI Gateway, browser, CDN response caching) — but that is a layer above storage, not the storage itself.

🎯 So — where is your page stored?
Nowhere, and everywhere. It exists as text inside code copied to every Cloudflare data centre. A request lands at the nearest one, a sandbox starts, your function returns that string, and the sandbox disappears. Nothing is kept for you. The code is replicated; the page is manufactured on demand and discarded. The only thing that genuinely sits still is your data — in Virginia.
