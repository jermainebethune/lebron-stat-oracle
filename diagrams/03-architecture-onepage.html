Request path — left to right, one question
1 Browser The question Two entry points: the custom domain for people, workers.dev for API clients.
2 Worker · access.js Checks input + who's asking Turnstile token, or an x-api-key header. Checked before any inference — this is the line where spending starts.
3 AI Gateway → Workers AI Writes SQL qwen2.5-coder-32b, temperature 0. Asked for a statement — never an answer. Repeat questions served from cache for 0 Neurons.
4 Guard Validates the SQL Allowlist with a preprocessing pass. Strips comments and literals first, then asserts one SELECT. 15 tests.
5 D1 Runs it, returns rows The only source of any figure the user sees. Nothing downstream can alter a number.
6 AI Gateway → Workers AI Describes the rows llama-3.2-3b. Only ever sees a non-empty result set, so it has nothing to get wrong.
7 Browser Answer + query + rows The prose never appears alone. The reader can always check it against its source.

Reading the colours

Model involved — output untrusted Ground truth — numbers originate Platform — Worker, routing Refusal — stops rather than guesses

Grounding a model in a datastore prevents invented numbers. It does not prevent a wrong query from producing a confidently wrong answer.

"How many MVPs did he win?" once returned 0 — the awards column existed but was empty, so the query succeeded and produced a real, meaningless figure. An empty column is more dangerous than a missing one, because an empty column answers. That is why five of the seven steps can refuse.
